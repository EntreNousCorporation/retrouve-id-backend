# Politique de confidentialité

**Dernière mise à jour : 13 avril 2026**

## 1. Responsable du traitement

RetrouvID est responsable du traitement des données collectées via
l'Application.

## 2. Données collectées

- **Identité** : prénom, nom, email, téléphone (si fourni).
- **Photo de profil** : facultative.
- **Déclarations** : type (perte/découverte), document, numéro partiel,
  nom figurant sur le document, photo (automatiquement floutée), lieu
  (ville, coordonnées GPS approximatives), date.
- **Messagerie** : contenu des messages échangés entre utilisateurs.
- **Notifications** : identifiants techniques pour push (OneSignal).

## 3. Finalités

- Créer et gérer le compte utilisateur.
- Rapprocher les déclarations de perte et de découverte (matching).
- Permettre la communication entre utilisateurs concernés.
- Envoyer des notifications relatives à votre activité.

## 4. Base légale

- Exécution du contrat (inscription, service de mise en relation).
- Consentement (notifications push, géolocalisation).

## 5. Destinataires

Les données ne sont partagées qu'entre utilisateurs concernés par une
mise en relation. Aucune cession commerciale.

Les prestataires techniques (OneSignal pour les notifications push,
fournisseur SMTP pour les emails OTP, hébergement européen) n'accèdent aux
données qu'en tant que sous-traitants.

## 6. Durée de conservation

- Déclarations actives : 90 jours, puis statut EXPIRED.
- Messages : conservés tant que les deux comptes existent.
- Données de compte : jusqu'à suppression par l'utilisateur.

## 7. Vos droits (RGPD)

- **Accès / portabilité (art. 15 & 20)** : téléchargez vos données depuis
  l'Application (Profil → Exporter mes données).
- **Rectification (art. 16)** : modifiez votre profil directement.
- **Effacement (art. 17)** : supprimez votre compte depuis l'Application
  (Profil → Supprimer mon compte). La suppression est immédiate et
  définitive, y compris photos et messages.
- **Opposition / limitation** : contactez-nous.

Pour toute demande : **privacy@ypy-it.com**

## 8. Sécurité

- Mots de passe hashés (BCrypt), communication HTTPS (TLS Let's Encrypt),
  JWT courte durée + refresh token, stockage chiffré des tokens côté mobile.

## 9. Cookies / traceurs

L'Application mobile n'utilise pas de cookies. Elle utilise l'IDFA/AAID
uniquement pour cibler les notifications push (avec votre accord).

## 10. Modifications

Toute évolution substantielle de la politique sera notifiée dans
l'Application.
